How the web works

  • Browser - A persons computer runs a browser which displays HTML files - Chrome,iE,Firefox etc
  • The internet - Is a network of computers
  • HTTP - Hypertext Transfer Protocol - how browser and server communicates
  • Servers - Host files that make up the web

HTML

  • Hyper Text Markup Language (HTML)
  • Element - This includes:
  • Tag
  • Contents
  • Closing Tag
  • Attributes - Some tags have attributes which are included in opening tag. For example
  • <a href="www.clockwise.co.nz">Click here for clockwise</a>
  • <img src="/images/picture.jpg" title="my picture"> - note no content = VOID TAG, so no closing tag

INLINE vs BLOCK

  • An inline tag will impact that immediate section
  • <b>, <em>, <span>
  • A block element creates an invisible box with width and height
  • <p>, <div>, <H1>

SPAN vs Div

  • Both have content
  • Span is an inline element
  • Div is block element
  • Enable style to be applied to them by adding a class
  • for example <div class="my-style">My content here</div>

HTML Document Structure

  • Doc Type - simple now HTML 5 <!Doctype html>
  • <html> tag and closing </html lang="en"> at the end of the page
  • Header - <head> - </head>
  • Page title - <title> the name of my web page </title>
  • Link to CSS - <link> rel="stylesheet" type="text/css" href="/css.mystylesheet.css" >
  • Link to Javascript - <script> type="text/javascript" src="http://www.mysite/blahblah.js"</script>
  • Body - <body> - </body>
  • This is where the majority of the page content will be

The Box Model

  • Content, padding and border are all used to calculate total size (ie 1+10+200+10+1 = 222)
  • This makes it hard to determine the actual size of an element
  • new model has been developed to make the total size equal the specified height or width
  • CSS:
  • -webkit-box-sizing;border-box;
    -moz-box-sizing;border-box;
    -ms-box-sizing;border-box;
    box-sizing;border-box;

Positioning boxes - Flexbox

  • DIVs are block elements so by default take up the full width
  • Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other.
  • This rule as-is works with Chrome 29+, IE 11+, and Mozilla 28+. In order to support Safari, the -webkit- prefix must be added:
    .app {
    display: -webkit-flex;
    display: flex;}
